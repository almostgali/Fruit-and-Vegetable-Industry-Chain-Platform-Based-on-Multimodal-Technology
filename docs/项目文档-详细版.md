# 项目文档（详细版）——智慧果蔬识别与物流管理平台

> 本文面向开发、测试与运维同学，覆盖架构、配置、数据库、API、前端交互、测试与部署、故障排查、后续规划等内容。建议与 `docs/项目文档.md` 一起使用，本文提供更细粒度的操作说明与示例。

---

## 一、项目概览
- 名称：智慧果蔬识别与物流管理平台
- 目标：打通“检测（品质/新鲜度）-下单-追踪-统计-展示”的全链路管理与可视化。
- 技术栈：
  - 后端：Spring Boot 3.1（Java 21）、MyBatis-Plus、MySQL 8、WebSocket、Thymeleaf
  - 前端：Thymeleaf、Axios、HTML/CSS/JS
  - 模型：YOLO（`fruit_quality_yolo.pt`、`fruit_freshness_yolo.pt`）
- 运行环境：Windows（提供批处理脚本），也可在 Linux/Mac 运行（需安装 JDK、Maven、MySQL）。

---

## 二、系统架构与模块
- Web 层：Controller（REST + 页面）+ WebSocket（大屏数据）
- Service 层：业务逻辑、参数校验、事务控制
- 数据访问层：MyBatis-Plus Mapper + XML（部分自定义查询）
- 前端：Thymeleaf 模板渲染 + Axios 调用 API；静态资源在 `static/css`、`static/js`

模块关系简述：
- 用户与会话：`UserController` 提供登录/注册；Session 用于物流管理员鉴权
- 模型检测：`ModelController` 接收图片上传调用 YOLO 模型，结果入库
- 物流管理：`LogisticsController` + `LogisticsManagementController` 提供订单增删改查、批量导入、统计与实时大屏
- 溯源追踪：`TraceabilityController` 提供创建、更新与轨迹查询

## 四、构建与运行
- 依赖与版本（摘自 `pom.xml`）：
  - Spring Boot：`spring-boot-starter-web`、`spring-boot-starter-thymeleaf`、`spring-boot-starter-jdbc`
  - MyBatis：`mybatis-spring`、`mybatis-plus-boot-starter`
  - WebSocket：`spring-boot-starter-websocket`
  - 数据库：`mysql-connector-j`
  - 其他：`lombok`
- 关键配置（`application.properties` 摘要）：
  - `server.port=8080`
  - `spring.datasource.url=jdbc:mysql://localhost:3306/fruit_recognition?useSSL=false&serverTimezone=UTC`
  - `spring.datasource.username` / `spring.datasource.password`
  - `mybatis-plus.mapper-locations=classpath:mapper/*.xml`
  - `spring.thymeleaf.prefix=classpath:/templates/`
  - 模型路径：`model.quality.path`、`model.maturity.path`
- 启动步骤：
  1. 安装并启动 MySQL 8，创建数据库 `fruit_recognition`
  2. 执行 `setup-database-fixed.sql` 或 `back/src/main/resources/sql/logistics_management.sql`
  3. 在 `back/` 下运行 `mvn spring-boot:run` 或双击 `start.bat`
  4. 浏览器访问 `http://localhost:8080`

## 六、后端控制器与 API
> 路径以 `/api` 开头的为 REST 接口；页面访问走非 `/api` 的 Controller。

- 用户与首页
  - `GET /` 或 `GET /index`：首页
  - `POST /user/login`：登录（表单或 JSON）
  - `POST /user/register`：注册
  - `GET /user/logout`：退出

- 模型检测
  - `POST /api/model/detect/quality`：果蔬品质检测（表单上传 `file`）
    - 返回：检测结果（类别、置信度）、存储路径、时间戳
  - （如有）`POST /api/model/detect/maturity`：新鲜度/成熟度检测

- 物流管理（订单/统计）
  - `GET /logistics/admin`：物流管理页面（Thymeleaf 模板 `logistics-management`）
    - Session：`logisticsAdmin` 存在时传入 `model`，用于页面展示
  - `POST /api/logistics/order`：创建订单（JSON `LogisticsOrder`）
  - `GET /api/logistics/order/list?page&size&city&status`：分页查询订单
  - `PUT /api/logistics/order/{orderId}/status?status=`：更新订单状态
  - `POST /api/logistics/order/batch`：批量创建订单（JSON 数组）
  - `GET /api/logistics/statistics`：统计数据（汇总、地区分布）
  - `GET /api/logistics/dashboard/realtime`：大屏实时数据（结合 WebSocket）

- 溯源追踪
  - `POST /traceability/create`：创建追踪记录
  - `POST /traceability/update`：更新追踪记录
  - `GET /traceability/track?orderNo=`：按运单号查询轨迹

### 请求示例（cURL）
- 创建订单
```
curl -X POST http://localhost:8080/api/logistics/order \
  -H "Content-Type: application/json" \
  -d '{
    "orderNo":"WL20241001-0001",
    "recipientName":"李四",
    "phone":"13800000000",
    "province":"江苏省",
    "city":"苏州市",
    "address":"工业园区星湖街",
    "itemWeight":0.8,
    "itemPrice":100,
    "freightPrice":12,
    "status":"CREATED"
  }'
```
- 订单列表
```
curl "http://localhost:8080/api/logistics/order/list?page=1&size=10"
```
- 更新状态
```
curl -X PUT "http://localhost:8080/api/logistics/order/1001/status?status=DELIVERED"
```
- 品质检测（上传图片）
```
curl -X POST http://localhost:8080/api/model/detect/quality \
  -H "Content-Type: multipart/form-data" \
  -F file=@test_apple.jpg
```

## 十二、常见问题与故障排查
- 订单列表加载失败：
  - 检查前端路径是否与后端一致（已修复为 `/api/logistics/order/...`）
  - 确认管理员登录态存在（Session）
- 8080 端口占用：修改 `server.port` 或释放占用进程
- 数据库连接失败：确认 `spring.datasource.*` 与 MySQL 服务
- 模型检测失败：检查模型路径与文件权限；确保 `python` 推理依赖满足（如有）
