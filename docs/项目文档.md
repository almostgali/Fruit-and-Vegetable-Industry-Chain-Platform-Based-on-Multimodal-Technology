# 项目文档（智慧果蔬识别与物流管理平台）

## 项目概述
- 项目名称：智慧果蔬识别与物流管理平台
- 目标：提供果蔬品质/新鲜度检测、用户登录、物流订单管理、溯源追踪与实时数据大屏展示，打通“检测-下单-追踪-统计”全流程。
- 技术栈：Spring Boot 3.1、Java 21、MyBatis-Plus、MySQL 8、Thymeleaf、WebSocket、Axios、HTML/CSS/JS。

## 系统架构
- 前端：Thymeleaf 模板（`templates/`），静态资源（`static/css`、`static/js`）。主要页面：`index`、`quality-detection`、`maturity-detection`、`traceability`、`logistics-management`。
- 后端：Spring Boot REST 控制器 + Service + MyBatis-Plus Mapper。
- 数据层：MySQL，MyBatis-Plus分页与XML映射（`resources/mapper`）。
- 实时通信：WebSocket（`/ws/logistics-dashboard`）推送大屏数据。

## 目录结构
- `back/src/main/java/org/back`：后端源码
  - `controller`：页面与API控制器（用户、模型检测、物流管理、溯源）
  - `service`、`service/impl`：业务逻辑实现
  - `entity`：实体模型（订单、管理员、追踪、统计）
  - `mapper`：MyBatis-Plus Mapper 接口
  - `config`：MyBatis Plus 和 WebSocket 配置
- `back/src/main/resources`
  - `templates`：Thymeleaf 页面模板（含 `logistics-management.html`、`traceability.html`）
  - `static/css`、`static/js`：静态样式与脚本（含 `logistics-management.css`、`logistics-management.js`）
  - `mapper`：XML 映射文件
  - `sql`：示例建表脚本 `logistics_management.sql`
- 根目录脚本：`start.bat`、`compile-project.bat`、`mysql-setup.bat`、`check-env.bat`

## 构建与运行
- 环境要求：
  - Java 21、Maven 3.9+、MySQL 8.x
  - Windows 平台提供启动脚本：`start.bat`
- 依赖：见 `back/pom.xml`
  - Spring Boot Web、Thymeleaf、JDBC、MyBatis-Spring、MyBatis-Plus、WebSocket、Lombok、MySQL Connector
- 配置：`back/src/main/resources/application.properties`
  - `server.port=8080`
  - `spring.datasource.*` 指向 `fruit_recognition` 数据库
  - 模型文件路径：`model.quality.path`、`model.maturity.path`
- 启动：
  1. 初始化数据库（运行 `mysql-setup.bat` 或导入 `setup-database-fixed.sql`/`logistics_management.sql`）
  2. 进入 `back` 目录，执行 `mvn spring-boot:run` 或 `start.bat`
  3. 打开 `http://localhost:8080`

## 主要功能与页面
- 用户登录/注册（`/user/login`、`/user/register`）：登录后访问首页 `/index`
- 果蔬质量检测（`/api/model/detect/quality`）与成熟度检测：支持图片上传并保存检测结果
- 溯源追踪（`/traceability`）：查询与创建物流追踪记录
- 物流管理（`/logistics/management` 或 `/logistics/admin`）：
  - 新增订单、订单列表、订单状态更新、批量录入
  - 数据统计与实时大屏（WebSocket）

## 后端API说明（节选）
- 基础路径：`/api/logistics`
  - `POST /order` 创建订单，`body` 为 `LogisticsOrder`
  - `GET /order/list?page&size&city&status` 分页查询订单列表
  - `PUT /order/{orderId}/status?status=` 更新订单状态
  - `POST /order/batch` 批量创建订单（List<LogisticsOrder>）
  - `GET /dashboard/realtime` 获取大屏实时数据
- 模型检测：`POST /api/model/detect/quality`（表单上传 `file`）
- 溯源追踪：`POST /traceability/track`、`POST /traceability/create`、`POST /traceability/update`

### 请求/响应示例
- 创建订单
  - 请求：`POST /api/logistics/order` JSON
  - 响应：`{ success: true, data: LogisticsOrder, message: "订单创建成功" }`
- 订单列表
  - 请求：`GET /api/logistics/order/list?page=1&size=10`
  - 响应：`{ success: true, data: { records, total, current, size, pages } }`

## 前端交互要点
- 页面 `logistics-management.html` 引入 `axios` 与 `/js/logistics-management.js`
- JS 初始化：`initializePage()`、`loadOrderList()`、`loadStatistics()`
- 订单表单 `#orderForm` 触发 `handleOrderSubmit`（已修复API路径与参数）
- 打开实时大屏：`openDashboard()` 指向 `/traceability`；刷新：`refreshDashboard()` 调用 `/api/logistics/refresh-dashboard`（可扩展）

## 数据库与表
- 关键表：`logistics_order`、`logistics_track`、`warehouse`、`warehouse_stock`、`logistics_statistics`
- Mapper：`LogisticsOrderMapper` 提供统计与分页查询；`LogisticsStatisticsMapper`；`LogisticsTrackingMapper.xml`

## 技术路线
- 后端：Spring Boot + MyBatis-Plus 实体/分页/自定义查询；REST API + WebSocket 推送；会话管理（HttpSession）
- 前端：Thymeleaf 模板渲染 + Axios 调用；页面模块化（订单管理、批量录入、统计、大屏）；UI 样式在 `logistics-management.css`
- 数据存储：MySQL 8，使用 SQL 脚本初始化与扩展

## 常见问题
- 订单加载失败：通常为前端路径与后端不一致或Session未登录导致。已修复并验证。
- 端口占用：确认 `server.port=8080` 未被其他服务占用。
- 数据库连接失败：检查 `spring.datasource.*` 配置及 MySQL 服务状态。
